# OpenTelemetry Example in Go

This repository contains a full-stack observability example using **OpenTelemetry** with a Go service. It demonstrates how to generate and collect Traces, Metrics, and Logs, and visualize them using the Grafana LGTM stack (Loki, Grafana, Tempo, Prometheus).

## Architecture

* **Go Application (`hello-service`)**: Instrumented with the OpenTelemetry Go SDK to emit traces, metrics, and logs over OpenTelemetry Protocol (OTLP).
* **OpenTelemetry Collector**: Receives the OTLP data, processes it, and routes it to the appropriate backend.
* **Prometheus**: Stores metrics.
* **Tempo**: Stores distributed traces.
* **Loki**: Stores logs.
* **Grafana**: Visualizes all the telemetry data.

## Getting Started

### 1. Start the Infrastructure

Use Docker Compose to spin up the OpenTelemetry Collector and the entire observability backend (Prometheus, Tempo, Loki, and Grafana):

```bash
docker compose up -d --build
```

### 2. Run the Go Service

Start the instrumented Go service locally:

```bash
go run main.go
```
*Note: The Go service will automatically send data to the OpenTelemetry collector running on `localhost:4317` (gRPC).*

### 3. Generate Telemetry Data

Send requests to the service's HTTP endpoint to generate traces, metrics, and logs:

```bash
curl http://localhost:8080/hello
```

### 4. View in Grafana

Open Grafana in your browser:
**URL**: [http://localhost:3000](http://localhost:3000)

* Authentication is disabled; you will be automatically logged in as an Admin.
* You can explore the data sources (Prometheus, Loki, Tempo) through the Grafana Explore tab to see the data generated by the `/hello` endpoint.

### 5. Querying Telemetry in Grafana

Navigate to the **Explore** tab in Grafana (the compass icon on the left sidebar) to query your telemetry data.

#### Traces (Tempo)
1. Select **Tempo** from the data source dropdown at the top left.
2. Under the **Search** tab, use the Service Name dropdown to select `hello-service`.
3. Click **Run Query** to see recent traces.
4. Click on a trace ID to see the waterfall view of spans (e.g., the `hello-handler` span). 

#### Metrics (Prometheus)
1. Select **Prometheus** from the data source dropdown.
2. In the query builder, switch to **Code** view and try querying the custom counter metric we created:
   ```promql
   hello_requests_count_total
   ```
   *Note: The OpenTelemetry SDK automatically appends `_total` to counter metrics.*
3. You can also filter by the custom attributes we added:
   ```promql
   hello_requests_count_total{endpoint="/hello"}
   ```

#### Logs (Loki)
1. Select **Loki** from the data source dropdown.
2. In the query builder, switch to **Code** view and enter a LogQL query to find logs from our service:
   ```logql
   {service_name="hello-service"}
   ```
3. To search for specific log messages (like our request handler logs):
   ```logql
   {service_name="hello-service"} |= "Handling hello request"
   ```
   *Note: Because `slog` logs are bridged via OpenTelemetry, they automatically include trace IDs. In Grafana, this allows you to seamlessly click on a log line and instantly jump to the corresponding distributed trace!*
